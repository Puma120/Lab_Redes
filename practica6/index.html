- Resumen -

En esta práctica, se creó un servidor local con apache para el cual acceder a una página web. Para ello, se ocupó XAMPP (Cross-Platform + Apache + MySQL + PHP + Perl) con su debida configuración y se probó el acceso al servidor con distintos dispositivos diferentes a la computadora que opera el programa.

- Introducción -

En el ámbito de las redes de computadoras, la configuración de una red local y el acceso a recursos compartidos son habilidades fundamentales para cualquier profesional en tecnología de la información. Una red local permite conectar múltiples dispositivos para compartir recursos como archivos, impresoras o servicios web, optimizando la eficiencia y la colaboración entre usuarios. En esta práctica de laboratorio, se busca conocer e identificar los pasos necesarios para configurar una red local básica, así como habilitar el acceso a un servidor web local utilizando herramientas como XAMPP. Este software permite montar un servidor Apache en una computadora, lo que facilita la creación y prueba de sitios web en un entorno controlado. Además, se explorará cómo configurar el firewall del equipo para permitir el acceso al puerto 80, identificar la dirección IP asignada al servidor y verificar el acceso al sitio web desde otros dispositivos conectados a la misma red. Esta práctica no solo refuerza conceptos teóricos sobre redes, sino que también desarrolla habilidades prácticas esenciales para la administración de sistemas y servicios en red.

- Desarrollo -

Instalación y Configuración

Antes de iniciar, es necesario descargar XAMPP de la página oficial. Después se requiere decidir qué contenidos se van a instalar. Para esta práctica solo es necesario Apache, el cual está seleccionado de manera predeterminada. A su vez, es de importancia ubicar (o seleccionar) la dirección de instalación, ya que se modificarán elementos de esta en un futuro. En nuestro caso, se decidió la ruta "D:\XAMMP" como punto de instalación.

Una vez instalado XAMPP, se puede abrir el programa para verificar su existencia en la computadora. Antes de levantar el servidor, primero se debe de acceder a la ruta en donde se instaló XAMPP en esta se encontrará una carpeta nombrada apache, dentro de esta en la carpeta conf existe un documento llamado httpd.conf (nuestra ruta equivalente es “D:\XAMMP\apache\conf\httpd.conf”), este debe de ser modificado. Se puede abrir con bloc de notas o con Visual Studio Code, en este alrededor de la línea 80 dirá Listen 80, esta línea se puede modificar dependiendo de la situación, en casos donde las IP son estáticas es recomendable obtener la IP del dispositivo a través de una terminal con el comando ipconfig dando el resultado que se ve en la siguiente imagen: .

Conociendo la IP de tu dispositivo y de otros dispositivos, es posible especificar el par de sockets para agilizar y facilitar al servidor su funcionamiento (para nuestro dispositivo sería Listen 172.18.0.86:80), a su vez evitando que dispositivos desconocidos puedan acceder. Pero debido a que no siempre se tendrá una IP estática, o no se desea cambiar de manera constante, el dejar Listen 80 o cambiarlo a Listen 0.0.0.0:80 permitirá a cualquier dispositivo dentro de la red acceder al contenido que ofrezca el servidor.

Se requiere a su vez configurar el firewall de su dispositivo para que permita las comunicaciones del servidor. En el caso de dispositivos de Windows esto se puede lograr a través de la opción de permitir la comunicación de aplicaciones, para ello se tendrá que agregar la ruta de la aplicación para httpd de apache (“D:\XAMMP\apache\bin\httpd”) y seleccionar si deseas que funcione en redes públicas y/o privadas.

- Conclusiones -

La realización de esta práctica permitió adquirir experiencia práctica en la configuración de una red local y la implementación de un servidor web utilizando XAMPP. A través de los objetivos específicos, se logró instalar y configurar un servidor Apache, habilitar el acceso al puerto 80 desde el firewall y verificar el funcionamiento del sitio web desde tres dispositivos distintos conectados a la misma red. Estos resultados resaltan la importancia de comprender tanto los aspectos técnicos como los procedimientos de seguridad necesarios para garantizar un acceso adecuado a los recursos en red. Además, esta actividad subraya la relevancia de las direcciones IP en la comunicación entre dispositivos y cómo estas pueden ser utilizadas para acceder a servicios locales. En conclusión, esta práctica no solo consolidó los conocimientos teóricos sobre redes locales, sino que también proporcionó las habilidades prácticas necesarias para configurar y administrar servidores en un entorno real.
