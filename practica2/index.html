ESP32 Sensores

- Resumen -

En esta práctica se explorará el uso de sensores digitales, analógicos e inteligentes utilizando el módulo NodeMCU ESP8266 o ESP32. Se configurará el entorno de desarrollo en Vokwi o Arduino IDE y se implementarán ejemplos para comprender el funcionamiento de un botón (sensor digital), un potenciómetro (sensor analógico) y un sensor DHT22 (sensor inteligente). Además, se desarrollará un programa para monitorear un sensor PIR, permitiendo detectar movimiento.

- Introducción -

Los microcontroladores ESP8266 y ESP32 son herramientas versátiles para la automatización y el desarrollo de sistemas embebidos, especialmente en aplicaciones que involucran comunicación WiFi e interacción con sensores. En esta práctica, se analizarán distintos tipos de sensores y su integración con el ESP32, clasificándolos en tres categorías:

-- Sensores digitales: Generan una señal binaria (0 o 1) en respuesta a una condición específica. Un ejemplo es un botón, que indica si está presionado o no.

-- Sensores analógicos: Producen una señal de voltaje variable proporcional a una magnitud física. En este caso, se usará un potenciómetro para simular una entrada analógica.

-- Sensores inteligentes: Incorporan un procesamiento interno para entregar datos preprocesados. Se utilizará el sensor DHT22, que mide temperatura y humedad.

-- Finalmente, se trabajará con el sensor PIR, que detecta movimiento mediante la variación de radiación infrarroja en su entorno.

- Desarrollo -

Antes de iniciar, es necesario configurar el entorno de desarrollo en Vokwi o Arduino IDE, según las indicaciones de la presentación y los tutoriales proporcionados.

Simulación

Antes de la implementación física, se utilizará la plataforma WOKWI para simular el comportamiento del circuito. La simulación permitirá verificar la lógica de encendido y apagado del LED, asegurando que el código funcione correctamente antes de cargarlo en el microcontrolador real. Se realizarán pruebas con diferentes condiciones, incluyendo el uso de botones para activar el LED mediante lógica digital con operadores OR y AND.

AND

Se definen tres pines: pinGPIO1 (salida). pinGPIO21 y pinGPIO47 (entradas). En la función setup(), se configuran los pines: pinGPIO1 como salida. pinGPIO21 y pinGPIO47 como entradas. Se inicializa pinGPIO1 en LOW. En el loop(), el programa lee los estados de las entradas (state1 y state2). Se evalúa la condición AND: Si ambas entradas están en HIGH, la salida pinGPIO1 se activa (HIGH). Si alguna o ambas entradas están en LOW, la salida permanece en LOW. Aplicaciones Simulación de circuitos digitales con compuertas lógicas. Control de dispositivos que requieren dos condiciones simultáneas para activarse.

OR

Este código implementa una compuerta lógica OR utilizando un ESP32. Se configuran dos pines (pinGPIO21 y pinGPIO47) como entradas y un pin (pinGPIO1) como salida. En cada iteración del loop(), el microcontrolador lee el estado de las entradas y, si al menos una de ellas está en nivel alto (HIGH), activa la salida estableciéndola en HIGH. En caso contrario, la salida permanece en LOW.

BOTÓN

Este código implementa un sistema en el que un botón controla el parpadeo de un LED. Se definen dos pines: gpio20, que funciona como entrada para leer el estado del botón, y gpio39, que actúa como salida para controlar el LED. En la función setup(), se configura gpio20 como entrada y gpio39 como salida. Luego, en el loop(), se lee el estado del botón y, si está presionado (HIGH), el LED parpadea con un retardo de 200 ms entre encendidos y apagados. Si el botón no está presionado (LOW), el LED permanece apagado.

LED ON

Este código establece un pin digital como salida y lo mantiene siempre en estado alto (HIGH). Primero, se define pinGPIO1 como el pin 1. En la función setup(), que se ejecuta una sola vez al inicio, se configura este pin como salida utilizando pinMode(pinGPIO1, OUTPUT). Luego, con digitalWrite(pinGPIO1, HIGH), se enciende el dispositivo conectado a este pin, como un LED. La función loop() está vacía, lo que significa que no hay cambios en el comportamiento después de la inicialización. El estado del pin permanecerá alto de manera indefinida.

LED OF

Este código configura un pin digital como salida y cambia su estado después de un tiempo determinado. Al inicio, se define pinGPIO1 como el pin 1. En la función setup(), se configura este pin como salida con pinMode(pinGPIO1, OUTPUT), y luego se establece en estado alto (HIGH), lo que activará el dispositivo conectado, como un LED. En la función loop(), que se ejecuta de forma continua, se espera 3 segundos (delay(3000)) y luego el pin se pone en estado bajo (LOW), apagando el dispositivo.

BLINK

Inicialmente, se define pinGPIO1 como el pin 1. En la función setup(), se configura como salida con pinMode(pinGPIO1, OUTPUT), y se inicia en estado bajo (LOW), asegurando que el dispositivo conectado comience apagado. En la función loop(), se ejecuta un ciclo donde: Se espera 500 ms (delay(500)). Se enciende la salida (digitalWrite(pinGPIO1, HIGH)). Se espera otros 500 ms. Se apaga la salida (digitalWrite(pinGPIO1, LOW)).

- Conclusiones -

A lo largo de esta práctica, se exploró el uso de entradas y salidas digitales en un microcontrolador para implementar diferentes funciones básicas de control. Se programaron circuitos que permitieron desde la activación de un LED con diferentes condiciones lógicas hasta la generación de parpadeos con retardos.

El primer conjunto de ejercicios demostró la implementación de compuertas lógicas mediante programación, utilizando operadores AND y OR para condicionar la activación de una salida en función de dos entradas digitales. Esto permitió observar el comportamiento lógico de estos operadores en un sistema físico, lo que es fundamental en el diseño de circuitos digitales.

Posteriormente, se implementó el control de un LED mediante la lectura de un botón, permitiendo generar un destello cuando el botón se presionaba. Este ejercicio ayudó a comprender la importancia del uso de retardos y la lectura de estados digitales en tiempo real.

Finalmente, se realizaron ejercicios enfocados en la generación de señales temporizadas, desde la activación y desactivación simple de un LED hasta el parpadeo continuo con diferentes intervalos de tiempo. Esto permitió reforzar el uso de retardos (delay) y su impacto en el comportamiento de un sistema embebido.

En conclusión, esta práctica brindó una introducción esencial a la manipulación de pines de entrada y salida en un microcontrolador, estableciendo las bases para el desarrollo de circuitos más complejos en aplicaciones de control digital y automatización.
