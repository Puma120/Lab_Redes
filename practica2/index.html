ESP32 Sensores

- Resumen -

En esta práctica se explorará el uso de sensores digitales, analógicos e inteligentes utilizando el módulo NodeMCU ESP8266 o ESP32. Se configurará el entorno de desarrollo en Vokwi o Arduino IDE y se implementarán ejemplos para comprender el funcionamiento de un botón (sensor digital), un potenciómetro (sensor analógico) y un sensor DHT22 (sensor inteligente). Además, se desarrollará un programa para monitorear un sensor PIR, permitiendo detectar movimiento.

- Introducción -

Los microcontroladores ESP8266 y ESP32 son herramientas versátiles para la automatización y el desarrollo de sistemas embebidos, especialmente en aplicaciones que involucran comunicación WiFi e interacción con sensores. En esta práctica, se analizarán distintos tipos de sensores y su integración con el ESP32, clasificándolos en tres categorías:

-- Sensores digitales: Generan una señal binaria (0 o 1) en respuesta a una condición específica. Un ejemplo es un botón, que indica si está presionado o no.

-- Sensores analógicos: Producen una señal de voltaje variable proporcional a una magnitud física. En este caso, se usará un potenciómetro para simular una entrada analógica.

-- Sensores inteligentes: Incorporan un procesamiento interno para entregar datos preprocesados. Se utilizará el sensor DHT22, que mide temperatura y humedad.

-- Finalmente, se trabajará con el sensor PIR, que detecta movimiento mediante la variación de radiación infrarroja en su entorno.

- Desarrollo -

Simulación

Antes de iniciar, es necesario configurar el entorno de desarrollo en Vokwi o Arduino IDE, según las indicaciones de la presentación y los tutoriales proporcionados.

PIR

En este código se espera leer de manera digital la entrada de un sensor de movimiento a base de infrarrojos. Para ello fue necesario determinar dos variables, una la cual guardaría la lectura del sensor y otra la cual determinaría si hay un movimiento en curso, se configuró entonces el GPIO 32 como salida, el GPIO 33 como entrada y se empezó la comunicación serial. Se lee entonces la señal del sensor Infrarrojo, si esta es alta significa que hay movimiento y se enciende el LED, si no existía un movimiento en curso se marca como que este empezó y se imprime el mensaje correspondiente. Cuando el movimiento pare (la lectura da baja) se checa si existía un movimiento, en caso de que si se apaga el LED y se manda el mensaje correspondiente.

Sensor inteligente

En este código se espera leer la entrada de un sensor inteligente capaz de determinar la temperatura y humedad (DHT22). Para ello fue necesario importar la biblioteca DHT.h, establecer una variable para la temperatura y otra para la humedad, configurar el GPIO 32 como salida, el GPIO 33 como entrada y empezar la comunicación serial. El cuerpo del código entonces espera 1 segundo, lee y guarda los valores de la humedad y temperatura, los imprime de manera serial y determina si el nivel de humedad es mayor o igual al 50% para encender o apagar un LED.

Sensor Analogico

En este código se espera leer analógicamente los valores de entrada provenientes de un potenciómetro. Para ello se decidió determinar el GPIO 33 como entrada, el GPIO 32 como salida e inicializar la comunicación serial en la configuración. El cuerpo del código entonces lee y guarda el valor analógico en una variable, lo imprime de manera serial y determina si este valor es mayor o menor al 50% del valor máximo posible (4095), si lo es el LED en GPIO 33 enciende de lo contrario se apaga.

Sensor Digital

En este código se espera leer una entrada digital en un GPIO y reflejar esta a través de una salida en otro. Para ello se eligió en la configuración el GPIO 33 de la ESP32 como entrada y el GPIO 32 como salida. Se lee entonces una entrada digital de manera constante (el estado de esta determinada por un botón) y se refleja en un led conectado al GPIO 32.

- Conclusiones -

Mediante esta práctica, se comprendió la diferencia entre sensores digitales, analógicos e inteligentes, así como su integración con el ESP32. Se observó cómo los sensores digitales trabajan con valores binarios, los analógicos proporcionan valores variables y los inteligentes incluyen procesamiento interno. Además, la implementación del sensor PIR permitió conocer su uso en detección de movimiento, lo que amplía sus aplicaciones en seguridad y automatización. Esta práctica refuerza los fundamentos de sensado y procesamiento de datos en sistemas embebidos con WiFi.
