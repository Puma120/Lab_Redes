ESP32 Sensores

- Resumen -

En esta práctica se explorará el uso de sensores digitales, analógicos e inteligentes utilizando el módulo NodeMCU ESP8266 o ESP32. Se configurará el entorno de desarrollo en Vokwi o Arduino IDE y se implementarán ejemplos para comprender el funcionamiento de un botón (sensor digital), un potenciómetro (sensor analógico) y un sensor DHT22 (sensor inteligente). Además, se desarrollará un programa para monitorear un sensor PIR, permitiendo detectar movimiento.

- Introducción -

Los microcontroladores ESP8266 y ESP32 son herramientas versátiles para la automatización y el desarrollo de sistemas embebidos, especialmente en aplicaciones que involucran comunicación WiFi e interacción con sensores. En esta práctica, se analizarán distintos tipos de sensores y su integración con el ESP32, clasificándolos en tres categorías:

-- Sensores digitales: Generan una señal binaria (0 o 1) en respuesta a una condición específica. Un ejemplo es un botón, que indica si está presionado o no.

-- Sensores analógicos: Producen una señal de voltaje variable proporcional a una magnitud física. En este caso, se usará un potenciómetro para simular una entrada analógica.

-- Sensores inteligentes: Incorporan un procesamiento interno para entregar datos preprocesados. Se utilizará el sensor DHT22, que mide temperatura y humedad.

-- Finalmente, se trabajará con el sensor PIR, que detecta movimiento mediante la variación de radiación infrarroja en su entorno.

- Desarrollo -

Simulación

Antes de iniciar, es necesario configurar el entorno de desarrollo en Wokwi o Arduino IDE, según las indicaciones de la presentación y los tutoriales proporcionados.

Escanear Wifi

En este código se espera visualizar que redes detecta la ESP32, para ello se ocupa la librería de wifi y se configura la comunicación serial, para poder escanear las redes wifi visibles para el dispositivo, las cuales serán impresas junto a un valor que representa la intensidad de la señal recibida.

Conectar Wifi

En este código se demuestra la capacidad de una ESP32 de conectarse a una red remota, para comprobar su correcto funcionamiento se ocupará la biblioteca de wifi y httpClient, la primera para configurar la conexión de la ESP32 a una red hotspot y las segunda para realizar una conexión a una página web y comprobar que se pueden realizar peticiones Get de manera exitosa de esta, comprobando que se tiene acceso a internet.

Thinkspeak Digital




Este código permite la lectura del estado de un botón conectado al pin 33 y el control de un LED en el pin 32, además de enviar el estado del botón a la plataforma ThingSpeak a través de una solicitud HTTP. Primero, establece una conexión Wifi según las credenciales otorgadas y luego, en el loop, lee continuamente el estado del botón: si está presionado, enciende el LED, y si no, lo apaga. Si la conexión Wifi está activa, envía el estado del botón a ThingSpeak mediante una petición GET con la clave API y el valor del sensor. La transmisión de datos ocurre cada 3 segundos y se muestra en el monitor serié junto con la respuesta del servidor.

Thinkspeak Analógico




Este código permite la lectura analógica de un potenciómetro conectado al pin 33 y el control de un LED en el pin 32, además de enviar el estado del botón a la plataforma ThingSpeak a través de una solicitud HTTP. Primero, establece una conexión Wifi según las credenciales otorgadas y luego, en el loop, lee continuamente el valor analógico del potenciómetro: si este es mayor al 50% del valor máximo, enciende el LED, y si no, lo apaga. Si la conexión Wifi está activa, envía el valor leído a ThingSpeak mediante una petición GET con la clave API y el valor del sensor. La transmisión de datos ocurre cada 1 segundos y se muestra en el monitor serial junto con la respuesta del servidor.

Thinkspeak Inteligente




Este código para ESP32 utiliza un sensor DHT22 para medir la temperatura y la humedad, y envía los valores de humedad a la plataforma ThingSpeak mediante una solicitud HTTP. Primero, establece una conexión Wifi con las credenciales necesarias y configura el sensor DHT22 en el pin 33. En el loop, cada segundo, lee los valores de temperatura y humedad, los muestra en el monitor serie y enciende un LED en el pin 32 si la humedad es mayor o igual al 50%. Si hay conexión Wifi, envía la humedad a ThingSpeak a través de una petición GET con la clave API y el valor del sensor.

Thinkspeak PIR




Este código para ESP32 utiliza un sensor PIR para detectar movimiento y envía la información a ThingSpeak mediante una solicitud HTTP. Primero, establece una conexión Wifi con las credenciales necesarias y configura el sensor PIR en el pin 33 y un LED en el pin 32. En el loop, cada segundo, lee el estado del sensor PIR: si detecta movimiento, enciende el LED e indica a través de comunicación serial que ha empezado un movimiento; si no, lo apaga e indica que el movimiento ha finalizado. Luego, si hay conexión Wifi, envía el estado del sensor (1 si hay movimiento, 0 si no) a ThingSpeak a través de una petición GET..

- Conclusiones -

Mediante esta práctica, se comprendió la diferencia entre sensores digitales, analógicos e inteligentes, así como su integración con el ESP32. Se observó cómo los sensores digitales trabajan con valores binarios, los analógicos proporcionan valores variables y los inteligentes incluyen procesamiento interno. Además, la implementación del sensor PIR permitió conocer su uso en detección de movimiento, lo que amplía sus aplicaciones en seguridad y automatización. Esta práctica refuerza los fundamentos de sensado y procesamiento de datos en sistemas embebidos con WiFi.
